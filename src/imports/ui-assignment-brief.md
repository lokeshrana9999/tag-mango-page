Front-End UI Assignment (Tagmango)
Project Brief
We’re excited to have you work on a UI development assignment for a page from our BackstagePass product.
This task is focused entirely on UI craftsmanship and visual quality. Your goal is to recreate the page as shown in the Figma design, with a strong emphasis on pixel-perfect execution, responsiveness, and attention to detail. No API integrations are required — the scope of this task is limited strictly to UI and layout.

Design Reference
Figma Design: BackstagePass – Product Design System

Please go through the Figma design carefully to understand the structure, layout, spacing, typography, colors, and overall look and feel of the page.

What You Need to Build
You are required to build the page shown in Figma, including the following UI components:
1. Feed Post Card UI
Build the feed post card exactly as shown in the design. Pay close attention to spacing, typography, and visual hierarchy.
2. Wrapper with Header
Create a wrapper layout around the feed post card and add the header as designed in Figma.
3. Subscriber Post Card with Confetti Header
Build the subscriber post card component based on the design.
The header should include a fun confetti-style visual or animation, implemented in a way that best matches the design intent.
4. Sidebar UI
Implement the sidebar with the following behaviors and states:
Blur states


Locked states


Day selection interaction


When clicking on different days, the white indicator should move accordingly. No additional sidebar-related logic is required beyond what is visually represented in the design.

Focus Areas
UI Development Only
This is a UI-focused assignment. You may use static or mock data where needed. No API connections or backend logic are required.
Mobile Responsiveness
The UI should be fully responsive and work smoothly across desktop and mobile screen sizes.
Light and Dark Mode
The page should support both light and dark modes, with appropriate color usage, contrast, and readability in each.
Visual Finesse
We will be paying close attention to:
Spacing and alignment


Typography and font hierarchy


Hover and active states


Smoothness of interactions


Overall polish and finish


High-quality visual execution will be a key factor in evaluation.

Before You Start Coding
Before jumping into development, please create a short plan outlining:
How you plan to structure and organize components


Your approach to building the different UI elements


How you will ensure responsiveness


How you plan to support light and dark modes



Development & Deployment
You may use any front-end tools, frameworks, or libraries you are comfortable with to build the UI.
Once complete, deploy the page on a publicly accessible URL (for example, using Vercel, Netlify, or a similar service).

Submission Details
Please share the following details via email:
Link to the planning doc


Deployed URL of the page


GitHub repository link




Evaluation Criteria
Your submission will be evaluated based on:
Accuracy compared to the Figma design


Responsiveness across devices


Light and dark mode implementation


Code structure and clarity


Overall UI polish and attention to detail


We encourage you to focus on building a clean, well-crafted interface that closely aligns with the provided design.
